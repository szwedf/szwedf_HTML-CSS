Service

ブリックワークスは、現場の「手が足りない」「属人化していて不安」を解消するために、
受託開発・SES（常駐支援）・運用保守/SRE・コンサルティングの4つの事業を展開しています。
単に人をアサインするのではなく、課題の整理から設計・実装・運用改善までを一気通貫で支援し、
お客様のプロダクトが「安心して伸ばせる状態」になることをゴールとしています。

特に、Web/業務システムの受託開発に強みを持ち、
要件定義〜UI/UX設計〜フロント/バックエンド実装〜テスト自動化〜クラウドインフラ構築までを
自社チームで完結できる体制を整えています。

システム開発 / 受託

Web / 業務システム、API連携、E2Eテスト自動化まで、
お客様のビジネスに合わせたシステムをフルスクラッチで開発します。

  • 要件ヒアリング・現状業務フローの棚卸し、課題の言語化
  • 画面遷移図・ワイヤーフレーム・UIデザインの作成
  • フロントエンド（Vue / React）・バックエンド（Laravel / Node.js / Python）の実装
  • REST / GraphQL API設計、外部SaaSや既存システムとの連携
  • 自動テスト・E2Eテスト・コードレビューによる品質担保
  • AWS / GCP 上でのインフラ設計・CI/CD パイプライン構築

例）社内業務のExcel管理をWebシステムに置き換えたい、
顧客向けの予約・会員・マイページシステムをゼロから作りたい、
既存PHPシステムをモダンな構成にリプレイスしたい、といったご相談が多い領域です。

SES（常駐支援）

既存チームにジョインし、「現場で一緒に手を動かす」エンジニアを提供します。
開発〜運用保守まで、長期的な伴走を前提とした支援スタイルです。

  • フロントエンド / バックエンドエンジニアとしての機能追加・改修作業
  • 要件定義〜設計会議への参加、技術的な打ち返し・見積もり支援
  • 運用チームと連携した障害調査・ログ解析・恒久対策の検討
  • ドキュメント整備、ナレッジ共有の仕組み化（Confluence / Notion など）
  • 内製化に向けたオンボーディング・教育・開発プロセス改善の提案

単なる「人月の補填」ではなく、現場のボトルネックを一緒に特定し、
中長期的にチームの生産性が上がる状態を目指します。

運用保守 / SRE

監視・アラート設計から Evidence 整備、Runbook作成、自動化までをトータルで支援。
「落ちない」「夜に鳴らない」システム運用を一緒につくります。

  • 監視項目・しきい値・アラートルールの設計（Zabbix / CloudWatch 等）
  • 障害対応フロー / エスカレーションルールの整理と可視化
  • Evidence・Runbook・手順書の整備と定期的なメンテナンス
  • バッチ処理・運用作業の自動化（スクリプト化 / ジョブ管理ツール連携）
  • SLO / エラーバジェットの設計、運用コストを意識した改善提案

「夜間アラートが多くてつらい」「誰か一人に運用が依存している」など、
現場のリアルな悩みベースで、設計〜運用ルールの整備まで並走します。

コンサルティング

要件定義・品質改善・開発プロセスの見直しなど、
「どこから手を付ければいいかわからない」状態から並走するライトな技術顧問プランです。

  • 現状システム・運用フローの棚卸しと課題整理ワークショップ
  • 技術選定、アーキテクチャのレビュー、リプレイス計画の検討
  • レビュー文化・テスト文化を根付かせるための仕組みづくり支援
  • 若手〜中堅エンジニア向けのメンタリング・勉強会の実施
  • 経営層・非エンジニア向けの「技術とビジネスをつなぐ」説明サポート

まずは月数時間からスタートし、
「受託開発 or SES だけでは届きにくい、上流のモヤモヤ」を一緒に整理していきます。
